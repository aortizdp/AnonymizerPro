The Core IdeaThe project was born from a specific need: How can we remove sensitive data from documents without sending that data to a third-party cloud?The Problem: Using Generative AI (like ChatGPT) for anonymization often involves sending raw, sensitive data to external servers, which violates many privacy regulations (GDPR, HIPAA).The Solution: A local, private tool that uses Non-Generative NLP to detect and mask Personally Identifiable Information (PII) entirely on the host machine.2. Project Planning & ArchitectureTo make the app professional and maintainable, we chose a Modular Architecture:Frontend (Streamlit): Handles the user interface, file uploads, and language switching.Backend (FastAPI): A high-performance API that processes the files using the NLP engine.Decoupled Configuration:config.py: Stores all external links (GitHub, BMC, Personal Web) to avoid "hardcoding" URLs.translations.py: A centralized dictionary for multi-language support (EN, CA, ES, FR, DE).3. Tech Stack: Selected LibrariesWe selected specific libraries based on a balance of performance, privacy, and ease of use:LibraryRoleWhy?Alternatives ConsideredStreamlitUI FrameworkFastest way to build data-centric web apps in Python.Flask or React (Too complex for this scope).FastAPIAPI EngineAsynchronous and extremely fast for file handling.Django (Too heavy/slow for simple APIs).Spacy/PresidioNLP CoreIndustry-standard for PII detection without internet access.NLTK (Less accurate for modern entities).PandasData ProcessingNecessary for handling .xlsx and .csv structures.Openpyxl (More limited for data analysis).python-docxWord SupportAllows the tool to parse and rebuild .docx files.PDFMiner (Only for PDFs).4. How the Tool Works (The User Guide)This tool follows a simple Two-Step Security Workflow:Step A: AnonymizationUpload a document (.xlsx, .csv, or .docx).The NLP engine identifies entities (Names, IDs, IBANs, etc.).The system replaces them with placeholders (e.g., USER_1, IBAN_A).Output: You receive a .zip containing the Anonymized File and a decryption_keys.xlsx file.Step B: Deanonymization (Restoration)Upload the Anonymized File.Upload the decryption_keys.xlsx file.The tool "translates" the placeholders back to their original values.Output: You get the original document restored.5.
